JavaScript中的易错点

  1. JavaScript中排序默认是字典序排序，需要按值升序则array.sort((a, b) => a - b);

  2. const和let只有块作用域，在代码块之外调用，声明let的那一行会报定义但未使用的错误，使用let的那一行会报未定义就使用的错误</br>

  3. JavaScript中数组的长度array.length后面没括号

  4. 如果解决算法是一个函数实现的，其中有涉及对全局变量的使用，那么使用之前一定要先初始化

  5. 如果回溯时push/pop的操作对象vari同时是需要返回的内容，那么上来最好先创建一个变量，let shinvari=vari.slice(); 并将push/pop操作的对象转换到shinvari

  6. 二维数组的初始化

    const shinArray = Array.from({length: 行数}, () => new Array(列数).fill(0));

    {length: 行数} 表示生成一个长度为目标行数的空数组

    new Array(列数)：为每一行创建一个长度为目标列数的新数组

  7. 对于JavaScript中需要取整的情况，可通过以下方式实现

  8. 当Map的键为数组时，慎用map.has(),这一方法比较的内容同样为引用

  9. 用new方法创建Map,Set时，记得Map,Set首字母是大写

  10. 未声明的变量使用后会自动转为var。涉及迭代的变量一定先let定义。执行结果中只有最后一项或缺少最后一项考虑变量不独立的原因

  11. for循环遍历,for后面的元素都在括号内

  12. 但凡涉及处理字符串，先判空

  13. JavaScript的数组不能用if(数组名)实现判空

  14. JavaScript字符串没有push和pop方法

流程控制
基本定义

continue:跳过当前循环的一轮，进入下一轮

break:直接中断当前循环

在二维循环中

如果目标位置是continue，从理论上看，这个流程控制语句将没有任何作用，因为其跳过的仅仅是内层for...of...的一轮，但很不巧，内层循环中没有任何操作；

如果目标位置是break，那么一旦count>=item[0]&&count<item[1],内层循环直接中断，进入外层循环的下一轮。但在这之前，还会执行

那么，有没有什么方法，在满足count>=item[0]&&count<item[1]时，直接进入外层循环，且内层循环后的两句不执行呢？答案是有。

在ES6标准中，我们可以采用标签控制语句，即在标签前加一个标签名称，然后用continue和break实现对这个标签指代的循环的控制。以下是用标签控制语句实现上述效果的方法。

JavaScript中的二维数组
  1. 如果需要判定JavaScript中的一个二维数组中的一个一维数组元素是否和另一个一维数组在内容上相等，则不能直接使用 二维数组.includes(一维数组)，正确的做法是二维数组.some(item=>JSON.stringify(item)==目标数组)
  2. 接收DFS结果的高维数组，不能直接push(低维数组)，否则传进高维数组的只是低维数组的引用，应当push（[...低维数组]）,或在每一轮DFS时，用slice复制参数表里的数组给新的变量，再操作这个新的变量

此外，addEventListener后的不少函数也是回调函数

map

map() 是数组的一个内置方法，用于对数组中的每个元素执行一个函数，并返回一个新数组。该新数组的元素是由该函数的返回值组成的。map() 方法不会修改原始数组，它会返回一个新的数组。例如

res.map(item => item.text);

接收数组中的每个元素 item，然后返回 item.text 的值。

this.HistoryData = history.map(item => ({ hisdata: item.answer }));

将 item 对象中的 answer 属性的值赋给新对象的 hisdata 属性。

({ hisdata: item.answer }) 表示返回一个新的对象，hisdata 属性的值来自 item.answer

元素排布

display: flex;

设置父元素的布局为 Flexbox 布局，使得它的直接子元素成为 Flex 容器，从而可以使用 Flexbox 的对齐属性。

align-items: center;：将 子元素 在容器的 垂直方向（交叉轴）上居中对齐。

justify-content: center;：将 子元素 在容器的 水平方向（主轴）上居中对齐。

justify-content: space-between;：将子元素在主轴上（通常是水平方向）均匀分布，第一个子元素靠左，最后一个子元素靠右，中间的元素平均分布。

计算属性

计算属性的计算结果会被缓存，只有当其依赖的数据发生变化时才会重新计算。普通属性每次被访问时，都会执行它们的定义逻辑，无论依赖的数据是否变化。它们不具备缓存机制，因此每次调用时都需要重新计算。

普通属性可以通过 data 或 methods 来定义。

在 data 中，普通属性通常是直接的数据存储。

在 methods 中，普通属性是函数，每次调用时都会执行。

V-model

作用：vue中负责监听用户输入，实现数据更新

作用机理：

  1. v-bind绑定value属性的值；
  2. v-on绑定input事件监听到函数中，函数会获取最新的值赋值到绑定的属性中；

注意：

   1. 无论输入的type为什么样，v-model绑的都是string类型，如果需要化成数字类型，需要`v-model.number`
   1. `v-model.lazy`得话，则需要当前输入完，提交之后才能实现同步

微信小程序

生命周期

应用生命周期（App()）

应用生命周期函数在整个小程序的生命周期中执行。

  • onLaunch(options)：小程序启动时触发，通常用来做一些初始化工作，如获取系统信息、登录等操作。
  • onShow(options)：小程序从后台进入前台时触发。
  • onHide()：小程序进入后台时触发。
  • onError(error)：小程序发生脚本错误或 API 调用失败时触发。

页面生命周期（Page()）

页面生命周期函数适用于页面组件，控制页面的生命周期。

  • onLoad(options)：页面加载时触发，通常用来初始化页面的数据。
  • onShow()：页面显示时触发，页面每次显示时都会调用。
  • onReady()：页面首次渲染完成后触发。
  • onHide()：页面隐藏时触发。
  • onUnload()：页面卸载时触发，通常用来做资源清理等。
  • onPullDownRefresh()：页面下拉刷新时触发。
  • onReachBottom()：页面触底时触发。
  • onShareAppMessage()：页面被分享时触发。
