github使用从0到1

git的基础命令及操作

git remote -v查看远程仓库

运行结果示例

git checkout 分支名切换到某个分支

如果该命令无效，不妨尝试git pull后再切换

运行结果示例

git status查看状态

该命令会显示当前处于哪个分支，与哪个分支保持一致

git branch查看当前所在分支名称

一些组合套路

新建一个分支

git checkout -b 新分支名以当前分支为基础新建分支

git push -u origin 新分支名推送新建分支时，在远端（也就是GitHub）新建此仓库

交代码

一定一定一定先pull再开始写!

git commit -m "注释内容"(提交到仓库区) 将暂存区中的所有内容提交到本地仓库。

git add .将工作区（Working Directory）中所有被修改、新增的文件添加到暂存区（Staging Area）

git push推送

创建一个新仓库，并把本地的代码传上去

git config user.name "用户名"

git config user.email "邮箱"

ssh-keygen -t rsa -C "邮箱名"

这一步记得先在某个目录下创建一个ssh用于承接公钥

复制gitsshkey.pub里的所有内容到GitHub的NEW SSH KEY
